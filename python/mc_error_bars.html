def draw_graph_with_rates(net, s0_in):
    G=nx.DiGraph()
    G.add_nodes_from(s0_in.keys())
    edges = []
    for node in net.node_names:
        for onode in net.node_dict[node].sends_to:
            edges.append((node, onode))
    G.add_edges_from(edges)
    theta = np.arange(0, 720, len(s0_in.keys()))
    pos = {}
    for i in range(len(s0_in.keys())):
        pos[s0_in.keys()[i]] = (np.cos(i), np.sin(i))
    pos['E'] = (0,0)
    pos['A'] = (0,-1)
    pos['B'] = (1, 0)
    pos['C'] = (0, 1)
    pos['D'] = (-1,0)
    pos['F'] = (1,-1)
    
    try :
        pos['E1'] = (4, -1)
        pos['A1'] = (4, -2)
        pos['B1'] = (5, -1)
        pos['C1'] = (4, 0 )
        pos['D1'] = (3, -1)
        pos['F1'] = (2, -2)
    except KeyError:
        pass
    val_map = {}
    for (key, val) in s0_in.iteritems():
        if val=='infected':
            val_map[key] ='r'
        else: 
            val_map[key] = '.85'
    values = [val_map.get(node, .25) for node in G.nodes()]
    a= nx.draw(G, pos=pos, node_color = values, arrows=True)
    return a

def get_pos_neg(res, nullfalse = p_no_attacker_attacker_exists, nulltrue = p_no_attacker_attacker_doesnt_exist):

    neg = []
    for i in range(4):
        pos.extend(res[i])
        neg.extend(res[i+4])
    pos = np.asarray(pos)
    neg = np.asarray(neg)
    pos = pos.reshape(800,1)
    neg = neg.reshape(800,1)
    nullfalse = np.asarray(nullfalse).reshape(800,1)
    nulltrue = np.asarray(nulltrue).reshape(800,1)
    pos1 = np.hstack((pos, nullfalse))
    neg1 = np.hstack((neg, nulltrue))

The distribution of the log likelihood with an attacker with only 1000 samples

Recall that our estimate of the likelihood also has a distirbution. That means, if we repeatedly estimate the same dataset, we will not get the same estimate. My claim is that if the distribution of the estimate is skewed (fat part to the left, skinny part to the right, I forget if this is positive or negative skew), our small sample ROC can see some troubles.

Consider 1 realization of network activity. Suppose we are interested in classifying that realization as either an infected or clean net based on a likelihood ratio threshold of 10. That is, if the log likelihood ratio \(\frac{P(\text{data} | \text{attacker})}{P(\text{data} | \text{ no attacker})}\) is less than 10, we consider the net not to be infected. If it is greater than 10, we classify the net as infected. Consider a case when there is an attacker. Suppose that if we could calculate the numeratpr without an approximation (i.e. closed form solution), the likelihood ratio is \(10.5\). Therefore, at a threshold of 10, we would classify that realization as a positive and we would be correct. That is, we found a true positive.

However, we do not (yet) have a closed form solution. That means that even though the exact value of the likelihood would lead to a correct classification, our estimate may be significantly far enough away from the exact value that we actually classify the positive as a negative. For example, suppose that due to variance in the Monte Carlo estimate, we actually estimate the likeligood ratio to be 9.5. This would lead to a false negative classification.

Now, if the distribution around the exact (i.e. closed form) value of the likelihood was symmetric and identical (I'm sure there is a more formal term for these two conditions and I'm sure there are looser assumptions that capture the same idea) for any realization of the data, this misclassification would be cancelled out. For example, consider another dataset with an attacker whose exact likelihood value is 9.5. If the distribution of the MC estimate for this dataset was symmetric and the same for the case when the exact likelihood is 10.5, then the probability of this being classified as a positive when the threshold is 10 is the same as the data with a likelihood of a 10.5 being classified as a negative. Therefore, there would be a "wash".

Now, suppose that instead of the distirbution being symmetric around the exact likelihood value, the median is actually less than the exact value. That means that for a dateset that has an exact likelihood of say 10.000000000001, more than half of the time it will be classified as a false negative (since the median is less than 10). However, if we were to raise the number of samples so that the MC approximation was always exactly the closed form value, then this realization of the data would always be classified as a true positive. Therefore we see that when we raise the number of samples, we get better classification performance for a single realization.
